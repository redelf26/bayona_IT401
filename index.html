What is System Administration?

System Administration is the field of IT that involves managing, configuring, and maintaining computer systems and networks. System administrators ensure that technology infrastructure runs smoothly, securely, and efficiently.

Key Responsibilities

  • Server installation, configuration, and maintenance
  • User account management and access control
  • Network configuration and troubleshooting
  • System monitoring and performance optimization
  • Backup and disaster recovery planning
  • Security implementation and maintenance
  • Software installation and updates
  • Documentation and policy creation

Learning Journey Timeline

Weeks 1-2: Fundamentals

Learn basic concepts, hardware understanding, and operating system basics

Weeks 3-6: Linux Administration

Master command line, file systems, user management, and services

Weeks 7-10: Windows Administration

Learn Active Directory, PowerShell, and Windows Server management

Weeks 11-12: Networking & Security

Network configuration, firewalls, and security best practices

Weeks 13-14: Cloud & Modern Tools

Cloud platforms, automation, and monitoring tools

Weeks 15-16: Project & Career Prep

Capstone project and career preparation

üêß Linux System Administration

Command Line Mastery

Essential commands every Linux administrator must know:

# File and Directory Operations ls -la # List files with details cd /path/to/directory # Change directory mkdir -p /path/to/dir # Create directory structure cp -r source destination # Copy files/directories mv source destination # Move/rename files rm -rf directory # Remove files/directories find /path -name "*.txt" # Find files grep "pattern" file # Search in files # System Information top # Process viewer ps aux # Process list df -h # Disk usage free -h # Memory usage uname -a # System information uptime # System uptime

User and Permission Management

# User Management sudo useradd -m username # Add user with home directory sudo usermod -aG group user # Add user to group sudo passwd username # Set user password sudo userdel -r username # Delete user and home directory # Permission Management chmod 755 file # Set file permissions chown user:group file # Change file ownership sudo su - username # Switch to user groups username # Show user groups
‚ö†Ô∏è Warning: Always be cautious with sudo commands. They can make system-wide changes that may affect system stability.

System Services and Processes

# Systemd Service Management sudo systemctl start service_name # Start service sudo systemctl stop service_name # Stop service sudo systemctl restart service_name # Restart service sudo systemctl enable service_name # Enable at boot sudo systemctl status service_name # Check service status sudo systemctl list-units --type=service # List all services # Process Management kill PID # Terminate process killall process_name # Kill all instances nohup command & # Run command in background jobs # List background jobs fg %1 # Bring job to foreground

Log Management

# System Logs sudo journalctl -f # Follow system logs sudo journalctl -u service_name # Service-specific logs tail -f /var/log/syslog # Follow syslog grep "error" /var/log/messages # Search for errors logrotate /etc/logrotate.conf # Rotate logs

PowerShell Fundamentals

PowerShell is the command-line shell and scripting language for Windows administration:

# Basic PowerShell Commands Get-Process # List running processes Get-Service # List all services Get-EventLog -LogName System # View system event log Get-WmiObject -Class Win32_ComputerSystem # System information Get-LocalUser # List local users Get-LocalGroup # List local groups # File Operations Get-ChildItem C:\ -Recurse # List files recursively Copy-Item source destination # Copy files Move-Item source destination # Move files Remove-Item path -Recurse # Delete files/folders

Active Directory Management

# Active Directory PowerShell Commands Import-Module ActiveDirectory # Load AD module # User Management New-ADUser -Name "John Doe" -SamAccountName "jdoe" Get-ADUser -Filter * # List all users Set-ADUser -Identity jdoe -Department "IT" Remove-ADUser -Identity jdoe # Group Management New-ADGroup -Name "IT_Staff" -GroupScope Global Add-ADGroupMember -Identity "IT_Staff" -Members "jdoe" Get-ADGroupMember -Identity "IT_Staff" # Computer Management Get-ADComputer -Filter * # List all computers Reset-ComputerMachinePassword # Reset computer account

Firewall Configuration

Linux (iptables/ufw)

# UFW (Uncomplicated Firewall) sudo ufw status # Check firewall status sudo ufw enable # Enable firewall sudo ufw allow 22/tcp # Allow SSH sudo ufw allow 80,443/tcp # Allow HTTP/HTTPS sudo ufw deny from 192.168.1.100 # Block specific IP sudo ufw delete allow 80 # Remove rule # iptables sudo iptables -L # List rules sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT # Allow SSH sudo iptables -A INPUT -j DROP # Drop all other traffic sudo iptables-save > /etc/iptables/rules.v4 # Save rules

Windows Firewall

# Windows Firewall PowerShell Commands Get-NetFirewallRule # List firewall rules New-NetFirewallRule -DisplayName "Allow SSH" -Direction Inbound -Protocol TCP -LocalPort 22 -Action Allow Remove-NetFirewallRule -DisplayName "Block App" Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True

User Access Control

# SSH Key Management ssh-keygen -t rsa -b 4096 # Generate SSH key pair ssh-copy-id user@server # Copy public key to server ssh -i ~/.ssh/private_key user@server # Connect with specific key # sudo Configuration sudo visudo # Edit sudoers file safely # Add user to sudoers: username ALL=(ALL:ALL) ALL
üí° Security Tip: Always use SSH keys instead of passwords for server access, and disable root login via SSH.

System Hardening Checklist

  • Keep systems updated with latest patches
  • Disable unnecessary services and ports
  • Configure strong password policies
  • Enable system logging and monitoring
  • Set up automated backups
  • Configure fail2ban for intrusion prevention
  • Enable SELinux/AppArmor (Linux)
  • Regular security audits and vulnerability scans
